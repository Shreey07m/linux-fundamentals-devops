# Chapter 1: What is an Operating System (OS)?

An Operating System (OS) is software that acts as an intermediary between the user (applications) and computer hardware.

## Real-Life Analogy
Customer → User  
Waiter → Operating System  
Kitchen → Hardware  

The OS coordinates access to resources so everything runs smoothly.

## Responsibilities
1. **Process Management** → `ps`, `top`, `kill`
2. **Memory Management** → `free -h`, `vmstat`
3. **File Management** → `ls`, `cp`, `rm`
4. **Device Management** → `lsblk`, `lsusb`
5. **Security** → `chmod`, `chown`, `sudo`
6. **Networking** → `ip addr`, `ping`, `ssh`
7. **User Interface** → GUI vs CLI

## Types of OS
- Batch OS  
- Time-Sharing OS (Linux is one)  
- Real-Time OS  
- Distributed OS  
- Network OS

Running `docker run nginx` → OS allocates CPU/RAM, reads image, creates network, launches container.
